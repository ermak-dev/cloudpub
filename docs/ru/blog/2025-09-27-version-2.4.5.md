CloudPub 2.4.5 добавляет возможности удаленного управления публикациями и отладки HTTP-трафика.

<!-- truncate -->

## Полноценное управление публикациями через Web

Теперь все операции с публикациями доступны из личного кабинета без доступа к компьютеру:

- Создание новых публикаций
- Удаление публикаций
- Запуск и остановка публикаций
- Обновление агентов

### Автоматическое обновление агентов

Агенты, установленные как **системные сервисы**, могут быть обновлены удаленно из личного кабинета.

Для обновления агента выберите действие "Обновить" в списке агентов в личном кабинете. Действие доступно если вышла новая версия и агент запущен как системный сервис.

:::info
Обычные агенты не поддерживают удаленное обновление, так как для установки требуются права администратора. В этом случае воспользуйтесь кнопкой "Обновить" в интерфейсе.
:::

### Установка агента как сервиса

#### Linux и macOS

```bash
sudo ./clo service install
sudo ./clo service start
```

На Linux сервис использует systemd, на macOS — launchd. Сервис автоматически запускается при загрузке системы.

После установки проверьте статус:

```bash
sudo clo service status
```

#### Windows

Запустите командную строку от имени администратора:

```cmd
clo service install
clo service start
```

Сервис регистрируется в Windows Service Manager и запускается автоматически при загрузке.

Проверить статус можно командой:

```cmd
clo service status
```

или через `services.msc`.

### Защита настроек

Агент, установленный как сервис, работает с правами системы и использует системный файл конфигурации. Локальные пользователи не могут изменить его настройки без прав администратора. Это предотвращает несанкционированное изменение параметров публикаций.

:::info
Подробнее о расположении файлов конфигурации и логов для сервисов см. [FAQ](/faq).
:::

## Проигрывание HTTP запросов из инспектора

Инспектор трафика теперь позволяет повторно отправить любой HTTP запрос на ваш локальный сервис. Это упрощает отладку: можно воспроизвести проблемный запрос без необходимости вручную формировать curl-команду или настраивать Postman.

![Повторный запрос из инспектора](/img/replay.png)

Запрос отправляется с теми же заголовками и телом, что были зафиксированы инспектором. Ответ отображается в реальном времени.

## SDK для Rust и Python

Добавлены официальные SDK для работы с CloudPub API на языках Rust и Python. SDK упрощают интеграцию CloudPub в ваши приложения и автоматизацию управления публикациями.

Подробнее см. [документацию по SDK](/sdk).
